# 🎾 앵그리 테니스 클럽 대진표 생성기

클럽 주간 모임의 복식 대진표를 자동으로 만들어주는 도구입니다.

## 바로 사용하기

**`dist/앵그리대진표.html` 파일을 더블클릭**하면 브라우저에서 바로 열립니다.
설치·인터넷 연결이 필요 없고, 휴대폰으로 파일을 보내서 열어도 동작합니다.

1. 모임 유형 선택 — **정기모임**(a·b코트 게임 + c코트 레슨), **게임데이**(3코트 전부 게임), 또는 **앵그리대회**(별칭 대진표 + 현장 제비뽑기)
2. 오늘 참석자 번호 선택 (남 1~9, 여 A~I — 번호가 빠를수록 실력 상위)
3. 필요하면 개인 특성(게임선호⚡/신규회원🔰/혼복선호💞)과 지각·조퇴 라운드 설정
4. **대진표 생성** — 마음에 안 들면 **다시 섞기**
5. 수동 조정: 대진표에서 선수 이름 두 개를 차례로 누르면 자리를 맞바꿈 (규칙 위반 시 경고 표시)
6. **인쇄** 버튼으로 출력

마지막에 선택한 참석자·특성은 브라우저에 저장되어 다음 주에 그대로 불러옵니다.

## 앵그리대회 모드 (별칭 대진표)

매분기 대회용. 참석자의 남/여 수만큼 **남1·남2…/여1·여2… 별칭으로 대진표를 미리 준비**하고, 당일 현장 제비뽑기로 각 별칭에 실제 멤버를 배정합니다.

- 별칭엔 랭킹이 없으므로 게임 점수차 상한·라이벌 라운드·혼복 위주·랭커 라운드 설정은 숨겨집니다 (규칙은 게임데이 기반: 3코트, 인당 게임 수, 인당 최소 혼복).
- 대진표 아래 **별칭 배정 패널**: 참석자별 별칭 드롭다운(동성만, 이미 배정된 별칭은 숨김 → 1:1 자동 중복방지). **🎲 제비뽑기** 버튼으로 무작위 배정(현장 추첨 대체/기록).
- **별칭 ↔ 실명 토글**로 대진표에 실제 이름을 반영해 볼 수 있습니다.
- 공유 링크 뷰어는 기본 별칭 표시, 배정을 마친 뒤 공유하면 뷰어에서도 실명 토글이 가능합니다.
- (경기 결과 입력·앵그리랭킹 누적은 추후 확장 예정.)

## 공유와 권한

데이터(명단·설정·대진표)는 각자 기기의 브라우저에 저장되며, 공유는 링크로 합니다.

- **📤 대진표 공유 링크** — 매주 단톡방에 올리는 용도. 받는 사람은 **보기 전용 화면**(대진표 + 인쇄)만 봅니다.
  링크에는 암호화된 명단·설정도 함께 담겨 있어, 비밀번호를 아는 관리자에게는 명단 전달 역할도 겸합니다.
- **관리자 비밀번호** (고급 설정에서 지정) — 링크 속 명단·설정은 이 비밀번호로 암호화(AES-GCM)됩니다.
  비밀번호를 아는 사람이 보기 화면에서 "🔑 관리자 모드"로 들어가면 명단·설정·대진표가 그 기기에 저장되어
  이후 직접 대진표를 만들 수 있습니다. 매주 대진표 링크를 열기만 해도 명단 백업이 자동 갱신됩니다.
- 비밀번호를 변경하면 이전 링크로는 명단을 불러올 수 없습니다(대진표 보기는 계속 가능).
- 참고: 대진표 자체(이름 포함)는 링크를 가진 누구나 볼 수 있습니다. 명단의 실력 순위·개인 특성만 암호로 보호됩니다.

## 대진 규칙 (우선순위 순)

1. 한 라운드에 1인 1게임, 게임코트는 항상 채움 **(하드)**
2. 잡복 금지 — 남복/여복/혼복만 **(하드)**
3. 같은 파트너 중복 0회 **(하드)**
4. 게임 수 균등(차이 ≤1), 2라운드 연속 결장 금지
5. 정기모임: 3라운드 내 전원 1회 레슨 로테이션
6. 게임데이·앵그리대회: **인당 게임 수 우선**(기본 켜짐) — 전원이 목표 게임 수를 채우도록 최우선 배정.
   덜 중요한 제약부터 자동 완화하고, 성비가 크게 치우쳐 불가피할 때만 잡복을 **최소 게임 수**로 허용(배너 표시). 끄면 규칙 우선(잡복 0, 게임 수 편차 가능). 인당 최소 혼복 게임 수도 설정(기본 1회)
7. 랭커 라운드(설정, 기본 2): 상위 랭커끼리 게임 — 남복/여복은 상위 5명 중 4명, 혼복(혼복 위주 라운드와 겹칠 때)은 남녀 각 상위 3명 중 2명을 매번 랜덤 선정
8. 라이벌 라운드(설정, 기본 1·2·3): 비슷한 실력 4명이 한 게임 (팀은 균형 분할)
9. 같은 상대 재대면 최소화(상한 설정, 기본 2번) + 팀 합산 점수 균형
10. 개인 특성 반영 (정기모임)
11. 게임 유형 — 혼복 위주 라운드(설정, 기본 1·3)는 혼복 위주, 나머지는 동성복식 위주. 게임데이는 동성복식 위주
12. 레슨 그룹 실력 유사성

지킬 수 없는 조합(예: 남자 1명, 12명 게임데이에 남자 홀수)은 이유와 대안을 안내하고,
불가피한 완화(연속 결장 등)는 파란 배너로 표시됩니다.

## 개발

순수 JavaScript(ESM), 빌드 도구·외부 의존성 없음. 개발에만 Node.js 18+ 필요.

```
npm test        # 8~18명 남녀 전 조합 + 엣지 케이스 자동 검증 (139 tests)
npm run build   # dist/앵그리대진표.html 단일 파일 생성
```

```
engine/
  scheduler.js   공개 API: generateSchedule(config)
  planner.js     라운드·코트 계획, 실현 가능성 사전 검사 (패리티/파트너 풀)
  construct.js   라운드별 구성: 결장자 선정 → 페어링 백트래킹 → 매치업
  optimize.js    사전식 비용 튜플 + 로컬 서치 (하드 제약 보존 스왑)
  validate.js    검증기 — 엔진 자기검증·UI 수동편집 경고·테스트 공용
  rng.js         시드 PRNG (같은 시드 = 같은 대진표)
app.js           웹 UI (index.html은 로컬 서버에서 모듈로 로드)
build.mjs        엔진+UI를 단일 HTML로 번들
test/            node --test (파일은 .mjs — 일부 윈도우 환경에서 한글 경로 .js 엔트리 크래시 회피)
```

## 앵그리랭킹 (경기 결과·순위)

앵그리대회 결과를 입력해 멤버별로 누적 랭킹을 매깁니다.

1. 앵그리대회 대진표 생성 → 제비뽑기로 별칭 배정 완료
2. 결과 대진표에서 **"📝 결과 입력"** → 각 게임에 스코어(예: 6 : 4) 입력
3. **"이 대회 결과 저장"** → 별칭이 실제 멤버로 확정되어 저장, **🏆 앵그리랭킹** 카드에 누적
4. 랭킹은 **승수 우선 → 득실차 → 득점** 순 (리그 순위표 방식). 종합점수 = 승수 × 3
5. **전체 누적(연말 시상)** 과 **대회별 단일(당일 시상)** 을 상단 "보기" 칩으로 전환.
   대회별 단일 랭킹에는 🥇🥈🥉 메달과 선수별 경기 스코어(예: `60 63 61 63`, 이긴 게임 초록)도 표시
6. 저장된 대회는 개별 삭제 가능(랭킹 자동 재계산), **백업 내보내기/가져오기**(JSON)로 이동·복원

현재는 이 기기(경기이사)에 누적됩니다. **Firebase 실시간 공유는 다음 단계**에서 추가 예정
(멤버들이 각자 기기에서 랭킹 조회). 게임데이 결과 집계도 추후 포함 가능.
